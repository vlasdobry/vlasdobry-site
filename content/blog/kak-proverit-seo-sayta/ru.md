**TL;DR:** Проверьте 8 базовых параметров: Title, Description, H1, robots.txt, sitemap.xml, HTTPS, мобильную версию и скорость загрузки. Это займёт 10 минут и покажет критические проблемы. Для глубокого анализа нужен полный аудит.

Клиенты часто спрашивают: "Как понять, всё ли в порядке с SEO на моём сайте?" Ответ простой — проверить базовые вещи можно самому, без специалиста. Покажу как.

## Что проверяем

Восемь параметров, которые влияют на индексацию и ранжирование. Если хотя бы один из них сломан — сайт теряет позиции.

### 1. Title (заголовок страницы)

**Что это:** текст во вкладке браузера и в результатах поиска.

**Как проверить:** откройте сайт, посмотрите на вкладку. Или в коде страницы найдите `<title>`.

**Что должно быть:**
- 50-60 символов
- Ключевое слово в начале
- Уникальный для каждой страницы

**Частые ошибки:**
- "Главная" или "Home" вместо описания
- Одинаковый Title на всех страницах
- Слишком длинный (обрезается в выдаче)

### 2. Meta Description

**Что это:** описание страницы в результатах поиска (серый текст под заголовком).

**Как проверить:** в коде страницы найдите `<meta name="description">`.

**Что должно быть:**
- 120-160 символов
- Содержит ключевое слово
- Призыв к действию или выгода

**Частые ошибки:**
- Отсутствует вообще
- Одинаковый на всех страницах
- Скопирован первый абзац текста

### 3. Заголовок H1

**Что это:** главный заголовок на странице.

**Как проверить:** в коде страницы найдите `<h1>`. Должен быть ровно один.

**Что должно быть:**
- Один H1 на страницу
- Содержит ключевое слово
- Отличается от Title

**Частые ошибки:**
- Несколько H1 на странице
- H1 отсутствует
- Логотип обёрнут в H1

### 4. robots.txt

**Что это:** файл с инструкциями для поисковых роботов.

**Как проверить:** откройте `вашсайт.ru/robots.txt`

**Что должно быть:**
- Файл существует (не 404)
- Разрешает индексацию нужных страниц
- Указывает на sitemap.xml

**Частые ошибки:**
- Файл отсутствует
- Блокирует весь сайт (`Disallow: /`)
- Блокирует важные разделы

### 5. sitemap.xml

**Что это:** карта сайта для поисковиков — список всех страниц.

**Как проверить:** откройте `вашсайт.ru/sitemap.xml`

**Что должно быть:**
- Файл существует
- Содержит все важные страницы
- URL актуальные (не 404)

**Частые ошибки:**
- Файл отсутствует
- Содержит устаревшие URL
- Не обновляется при добавлении страниц

### 6. HTTPS

**Что это:** защищённое соединение (замок в адресной строке).

**Как проверить:** посмотрите на адресную строку — должно быть `https://`, не `http://`.

**Что должно быть:**
- Весь сайт на HTTPS
- Нет "mixed content" (картинки по HTTP)
- Редирект с HTTP на HTTPS

**Частые ошибки:**
- Сайт на HTTP
- Часть ресурсов загружается по HTTP
- Нет редиректа

### 7. Мобильная версия

**Что это:** как сайт выглядит на телефоне.

**Как проверить:** откройте сайт на телефоне или используйте [Google Mobile-Friendly Test](https://search.google.com/test/mobile-friendly).

**Что должно быть:**
- Текст читается без масштабирования
- Кнопки достаточно большие
- Нет горизонтальной прокрутки

**Частые ошибки:**
- Десктопная версия на мобильном
- Слишком мелкий текст
- Элементы налезают друг на друга

### 8. Скорость загрузки

**Что это:** время до полной загрузки страницы.

**Как проверить:** [PageSpeed Insights](https://pagespeed.web.dev/) от Google.

**Что должно быть:**
- Оценка 50+ для мобильной версии
- LCP (загрузка главного контента) < 2.5 сек
- Нет критических ошибок

**Частые ошибки:**
- Тяжёлые картинки без сжатия
- Много внешних скриптов
- Нет кэширования

## FAQ

**Можно ли продвинуть сайт только этим чеклистом?**
Нет. Это минимум для индексации. Для роста позиций нужна работа с контентом, ссылками и поведенческими факторами.

**Как часто проверять?**
После каждого крупного обновления сайта. Или раз в месяц для профилактики.

**Что делать, если нашёл ошибки?**
Исправлять по приоритету: сначала robots.txt и sitemap.xml (влияют на индексацию), потом Title и Description (влияют на CTR), потом остальное.
