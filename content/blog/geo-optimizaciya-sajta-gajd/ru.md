**TL;DR:** GEO-оптимизация сайта — это 5 конкретных шагов: создать llms.txt, добавить Schema.org, написать FAQ, усилить E-E-A-T и проверить доступность для AI-ботов. Ниже — пошаговый чеклист с примерами кода.

Если вы уже знаете [что такое GEO](/blog/chto-takoe-geo-optimizaciya/) и зачем это нужно, пора переходить к делу. Эта статья — практический гайд. Никакой теории, только конкретные действия.

Я собрал всё, что проверяю при GEO-аудитах клиентских сайтов, в один чеклист. Пять шагов — от самого простого к самому сложному.

## Шаг 1. Создайте llms.txt

Файл `llms.txt` — точка входа для AI-систем. Это как визитка вашего сайта, только для ChatGPT, Claude и Perplexity. Без него AI-системы парсят весь сайт и часто промахиваются с выводами.

Положите файл в корень сайта: `https://yoursite.com/llms.txt`

Что писать:

```
# Название компании

> Короткое описание в 1-2 предложения

## Услуги
- Услуга 1: описание
- Услуга 2: описание

## Контакты
- Сайт: https://yoursite.com
- Телефон: +7 (XXX) XXX-XX-XX
- Email: info@yoursite.com

## Ключевые факты
- 10+ лет на рынке
- 200+ клиентов
- Работаем по всей России
```

Три правила:
1. **Только ASCII.** Никаких эмодзи, специальных символов. AI-парсеры могут споткнуться.
2. **Факты, не маркетинг.** "200+ клиентов" — хорошо. "Лучшая компания на рынке" — бесполезно.
3. **Обновляйте.** Добавили услугу — обновите llms.txt.

Продвинутый вариант — создать ещё `llms-full.txt` с полным контентом сайта в Markdown. Это рекомендация из спецификации llms.txt: базовый файл для быстрого обзора, полный — для глубокого анализа.

## Шаг 2. Добавьте Schema.org разметку

Schema.org — это язык, который AI понимает без интерпретации. Вместо того чтобы гадать, что значит текст на странице, система читает структурированные данные.

### Для локального бизнеса

Минимум — `LocalBusiness` на главной странице:

```json
{
  "@context": "https://schema.org",
  "@type": "LocalBusiness",
  "name": "Название",
  "description": "Описание",
  "url": "https://yoursite.com",
  "telephone": "+7-XXX-XXX-XX-XX",
  "address": {
    "@type": "PostalAddress",
    "addressLocality": "Город",
    "addressCountry": "RU"
  }
}
```

### Для услуг

Добавьте `ProfessionalService` или `Service`:

```json
{
  "@type": "ProfessionalService",
  "name": "SEO-аудит сайта",
  "description": "Полный аудит по 7 направлениям",
  "provider": {
    "@type": "Person",
    "name": "Имя Фамилия"
  }
}
```

### Для статей блога

`Article` + `FAQPage` — это комбо, которое AI-системы любят:

```json
{
  "@type": "Article",
  "headline": "Заголовок статьи",
  "author": {
    "@type": "Person",
    "name": "Автор"
  },
  "datePublished": "2026-02-22"
}
```

На практике я вижу, что сайты с полной Schema.org разметкой появляются в ответах AI на 40-60% чаще, чем без неё.

## Шаг 3. Напишите FAQ для AI

Обычный FAQ на сайте и FAQ для AI — разные вещи. AI-системам нужны:

**Конкретные вопросы, которые люди задают ChatGPT.** Не "Какие у вас условия?", а "Сколько стоит SEO-аудит сайта?" или "Как долго длится GEO-оптимизация?".

**Прямые ответы в первом предложении.** AI часто берёт именно первое предложение ответа. Сравните:

- Плохо: "Стоимость наших услуг зависит от множества факторов..."
- Хорошо: "SEO-аудит стоит от 30 000 рублей. Цена зависит от размера сайта."

**Schema.org FAQPage разметка.** Без неё FAQ — просто текст. С ней — структурированные данные, которые AI может цитировать дословно.

```json
{
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Сколько стоит GEO-оптимизация?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Бесплатная диагностика — 0 ₽. Полный GEO-аудит — 50 000 ₽."
      }
    }
  ]
}
```

Совет: спросите ChatGPT о вашей нише. Посмотрите, какие вопросы он задаёт уточняющими. Это и есть ваши FAQ.

## Шаг 4. Усильте E-E-A-T

E-E-A-T (Experience, Expertise, Authoritativeness, Trustworthiness) — не просто критерий Google. AI-системы тоже оценивают, можно ли доверять источнику.

Что конкретно делать:

**Страница "Об авторе" с фактами.** Не "я люблю своё дело", а "10 лет опыта, 50+ проектов, сертификаты X и Y". Чем больше проверяемых фактов, тем выше доверие AI.

**Контакты на каждой странице.** Телефон, email, мессенджеры. Сайт без контактов AI воспринимает как менее надёжный.

**Упоминания на внешних ресурсах.** Публикации на отраслевых площадках, профили на LinkedIn, GitHub, профессиональных каталогах. AI проверяет, существует ли автор за пределами своего сайта.

**Конкретные кейсы с цифрами.** "Увеличили трафик клиента на 200%" — проверяемое утверждение. "Мы профессионалы" — нет.

## Шаг 5. Проверьте доступность для AI-ботов

Последний шаг — убедиться, что AI-системы вообще могут прочитать ваш сайт.

### robots.txt

Откройте файл `robots.txt` и проверьте, не заблокированы ли AI-боты:

```
User-agent: GPTBot
Allow: /

User-agent: ClaudeBot
Allow: /

User-agent: PerplexityBot
Allow: /
```

Если вы видите `Disallow: /` для этих ботов — AI-системы вас не видят. Это осознанный выбор или ошибка?

### Скорость загрузки

AI-краулеры, как и обычные боты, уходят с медленных сайтов. Core Web Vitals влияют и на GEO.

### JavaScript-рендеринг

Если контент подгружается через JavaScript, AI-боты могут его не увидеть. Проверьте: откройте страницу с отключённым JS. Видно ли основной контент? Если нет — добавьте `<noscript>` fallback или серверный рендеринг.

| Шаг | Что сделать | Приоритет |

| 1 | llms.txt в корне сайта | Высокий |

Первые три шага можно сделать за один день. Серьёзно. Файл llms.txt пишется за 20 минут, Schema.org копируется из примеров выше, FAQ — это вопросы, которые вам и так задают клиенты.

## FAQ

**Можно ли сделать GEO-оптимизацию самостоятельно?**
Да, базовые шаги (llms.txt, robots.txt, FAQ) можно выполнить самостоятельно за 1-2 дня. Для Schema.org и глубокой оптимизации может понадобиться помощь разработчика.

**GEO-оптимизация работает для всех ниш?**
Лучше всего работает для локального бизнеса (отели, клиники, рестораны) и профессиональных услуг (юристы, маркетологи, разработчики). Для e-commerce с тысячами товаров подход другой.

**Как проверить, появился ли мой сайт в ответах ChatGPT?**
Задайте ChatGPT вопрос о вашей нише в вашем регионе. Например: "Какой отель выбрать в Сочи?" или "Где заказать SEO-аудит?". Если вас нет в ответе — пора начинать GEO-оптимизацию.
