# 168-ФЗ Compliance Checker — Design Document

**Дата:** 2026-02-25
**Статус:** approved
**Автор:** Vlas + Claude

## Позиционирование

Бесплатный инструмент проверки сайта на соответствие закону 168-ФЗ о защите русского языка. PR-инструмент и лид-магнит для SEO-аудита. Не отдельная услуга — воронка ведёт к `/services/seo/`.

Первый на рынке бесплатный онлайн-чекер по 168-ФЗ.

## URL

- `/168-fz` (RU)
- `/en/168-fz` (EN)

## UX-путь пользователя

1. Заголовок + краткое описание закона (2-3 предложения)
2. Поле ввода URL → кнопка «Проверить»
3. Анимация сканирования
4. Результат: скор + полный список элементов по категориям (сворачиваемые секции)
5. CTA: pre-filled Telegram с доменом и количеством проблем

## Категории проверяемых элементов

### Критические (вес: -10 баллов)

| Категория | HTML-элементы | Пример |
|-----------|--------------|--------|
| Кнопки | `button`, `a.btn`, `input[type=submit]`, `input[type=button]` | "Book Now", "Submit" |
| Заголовки | `h1`-`h3` | "Deluxe Suite", "Our Services" |

### Важные (вес: -7 баллов)

| Категория | HTML-элементы | Пример |
|-----------|--------------|--------|
| Навигация | `nav a`, `header a` | "About Us", "Contacts" |
| Формы | `label`, `input[placeholder]`, `select option` | "Check-in date", "Adults" |

### Средние (вес: -5 баллов)

| Категория | HTML-элементы | Пример |
|-----------|--------------|--------|
| Подзаголовки | `h4`-`h6` | "Room amenities" |
| Политики/ссылки | `footer a` со словами policy/terms | "Privacy Policy" |

### Информационные (не влияют на скор)

| Категория | HTML-элементы | Пример |
|-----------|--------------|--------|
| Мета-информация | `title`, `meta[name=description]` | "Best Hotel in Moscow" |

Мета-теги выносятся в секцию «Рекомендации», не учитываются в скоре (серая зона по 168-ФЗ — потребитель не видит их на странице).

## Логика детекции

### Извлечение текста
- Worker фетчит HTML целевого сайта
- HTMLRewriter (Cloudflare) обходит целевые элементы потоково
- Извлекает видимый текст (innerText), placeholder, value
- Дедупликация вложенных элементов: если `<nav><a><span>Book</span></a></nav>`, считать один раз (текст из `<a>`, не из `<span>` и `<nav>`)

### Детекция латиницы
- Regex: `/[a-zA-Z]{2,}/` — минимум 2 латинских символа подряд
- Применяется к каждому извлечённому тексту

### Whitelist (исключения)

Вынести в отдельный конфиг для удобного дополнения.

**Категории исключений:**
- **Технические:** URL, email-адреса, домены
- **Общепринятые аббревиатуры:** Wi-Fi, SMS, GPS, QR, PDF, USB, ID, TV, VIP, DJ, PR, IT, HR, KPI, CEO, SPA, OK, vs, FAQ
- **Числа и единицы:** любые строки, состоящие только из цифр + латинских букв-единиц (24h, 5km)
- **Одиночные символы:** одна латинская буква (напр. категория "A")

Whitelist расширяется по мере получения обратной связи от пользователей.

### Детекция SPA
Если `<body>` содержит только `<div id="root">` / `<div id="app">` и менее 3 текстовых элементов — показать сообщение: «Сайт использует JavaScript-рендеринг. Автоматическая проверка недоступна. Попробуйте проверить конкретные страницы или напишите мне для ручной проверки.»

## Скоринг

```
score = 100
for each finding:
  if finding.severity == 'critical': score -= 10
  if finding.severity == 'important': score -= 7
  if finding.severity == 'medium':   score -= 5
score = Math.max(0, score)
```

Цвета:
- 80-100: зелёный (соответствует)
- 40-79: жёлтый (есть замечания)
- 0-39: красный (не соответствует)

## Отображение результата

### Сводка (всегда видна)
```
Ваш сайт: 35/100 по 168-ФЗ
Найдено 13 элементов, возможно требующих перевода
```

### Детали (сворачиваемые секции по категориям)
```
▼ Кнопки (2 элемента) — критично
  ⚠ Кнопка бронирования: "Book Now"
  ⚠ Кнопка отправки формы: "Submit"

▼ Навигация (3 элемента) — важно
  ⚠ Ссылка в меню: "About Us"
  ⚠ Ссылка в меню: "Contacts"
  ⚠ Ссылка в меню: "Gallery"

▶ Заголовки (4 элемента) — критично
▶ Формы (2 элемента) — важно
▶ Политики (2 элемента) — незначительно

ℹ Рекомендации
  Title страницы: "Best Hotel in Moscow" — рассмотрите перевод
```

Каждый элемент показывается с **человекопонятным описанием** («Кнопка бронирования», «Ссылка в меню»), а не с техническим селектором.

### Формулировки
- Использовать «возможно, требует перевода» — не «требует перевода» (не является юридической экспертизой)
- Severity-метки: «критично», «важно», «незначительно»

## Дисклеймер

> Экспресс-проверка на наличие иностранных слов. Не является юридической экспертизой. Инструмент не проверяет наличие слов в нормативных словарях и регистрацию товарных знаков — эти исключения могут быть допустимы по закону 168-ФЗ. Для полной проверки обратитесь к юристу.

## CTA после результата

Динамический текст по баллам:

| Скор | Текст (RU) | Действие |
|------|-----------|----------|
| 80-100 | «Ваш сайт в порядке! Проверьте техническое SEO» | Ссылка на Health Score (`/services/seo/`) |
| 40-79 | «Есть что исправить. Нужна помощь?» | Telegram |
| 0-39 | «Сайт не соответствует 168-ФЗ. Исправим?» | Telegram |

Pre-filled Telegram:
```
Привет! Проверил сайт {domain} на 168-ФЗ — {count} элементов на латинице ({score}/100). Поможете с исправлениями?
```

## Архитектура

### Worker (backend)
- Новый endpoint в существующем `health-score-proxy` Worker
- Route: `POST /api/compliance`
- Request: `{ url: string }`
- Response: `{ score, findings[], meta, isSPA, error? }`
- Finding: `{ text, category, severity, description, element }`
- HTMLRewriter для потокового парсинга
- Whitelist в отдельном файле-конфиге
- SSRF-защита (как в Health Score)

### Frontend
- Новый компонент: `ComplianceChecker.tsx`
- HTML entry points: `168-fz.html`, `168-fz-en.html`
- i18n: тексты в `ru.ts`, `en.ts`
- Postbuild: `168-fz.html` → `/168-fz/index.html`, `168-fz-en.html` → `/en/168-fz/index.html`
- Vite: добавить в `rollupOptions.input`

### Трекинг (Яндекс.Метрика)
- `compliance_start` — запуск проверки
- `compliance_complete` — завершение с результатом
- `compliance_error` — ошибка
- `compliance_cta_click` — клик по CTA

## Что НЕ входит в MVP

- Проверка по нормативным словарям (требует базу данных)
- Проверка товарных знаков в Роспатенте (нет публичного API)
- Рендеринг SPA-страниц (требует headless browser)
- Email-отчёт / PDF-выгрузка
- Сохранение истории проверок (кроме localStorage последних 5 URL)
